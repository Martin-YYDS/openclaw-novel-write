# /novel tasks - 分解任务清单

## 触发方式

```
/novel tasks
```

或对话式：
```
分解任务清单
生成写作任务
```

---

## 目标

将创作计划分解为具体的、可执行的任务清单，**按卷分拆，细化到每一章节，含字数要求**。

---

## 执行步骤

### 1. 加载计划文档

读取：
- `stories/<story-name>/specification.md` - 章节规划
- `stories/<story-name>/creative-plan.md` - 创作计划

### 2. 生成任务列表

输出到 `stories/<story-name>/tasks.md`

---

## 任务模板（按卷分拆）

### 格式

```markdown
# [故事名称] - 任务清单

## 第1卷：[卷名]（约X章）

### T001 - 第1章：[章节标题]
- **状态**: pending
- **字数要求**: 2800-3200字
- **核心任务**: [本章主要任务]
- **关键情节**: [具体情节点]
- **涉及线索**:
  - PL-01([线索名]) ⭐⭐⭐ 主推进
  - PL-02([线索名]) ⭐ 背景
- **交汇点**: [如适用] X-001([交汇点描述])
- **伏笔埋设/揭晓**: [如适用] F-001埋设 / F-002揭晓
- **必须包含**: [关键元素列表]
- **章末钩子**: [悬念设置]
- **依赖**: 无
- **输出**: `content/volume1/chapter-001.md`

### T002 - 第2章：[章节标题]
- **状态**: pending
- **字数要求**: 3300-3700字
- **核心任务**: [本章主要任务]
- **关键情节**: [具体情节]
- **涉及线索**:
  - PL-01(身份秘密) ⭐⭐ 辅助
- **伏笔埋设**: F-001埋设([伏笔内容])
- **必须包含**: [关键元素]
- **章末钩子**: [悬念]
- **依赖**: T001(第1章)
- **输出**: `content/volume1/chapter-002.md`

---

## 第2卷：[卷名]（约X章）

### T010 - 第10章：[章节标题]
- **状态**: pending
- **字数要求**: 3800-4200字
- **核心任务**: [本章主要任务]
- **关键情节**: [具体情节]
- **涉及线索**:
  - PL-01(身份秘密) ⭐⭐⭐ 主推进
  - PL-03(隐藏势力) ⭐⭐ 辅助
- **交汇点**: X-001(关键转折)
- **伏笔揭晓**: F-001揭晓([揭晓内容])
- **必须包含**: [关键元素]
- **章末钩子**: [悬念]
- **依赖**: T009(第9章)
- **输出**: `content/volume2/chapter-010.md`
```

---

### 3. 任务统计

```markdown
## 任务统计

| 项目 | 数值 |
|------|------|
| 总章节数 | 30 |
| 总卷数 | 3 |
| 预估总字数 | 90,000-105,000字 |
| 预估完成时间 | 6-8周（每日1章） |

### 里程碑

- 第5章完成 → 第一卷完成
- 第15章完成 → 第二卷完成
- 第30章完成 → 全书完成
```

---

### 4. 优先级说明

- **P0**：必须完成，否则故事不完整
- **P1**：重要，建议完成
- **P2**：可选，增强体验

---

### 5. 依赖关系

- 大部分章节有依赖（需按顺序完成）
- 部分支线任务可并行

---

## 输出

文件路径：`stories/<story-name>/tasks.md`

---

## 提示下一步

任务清单创建完成后，可以开始执行：
```
/novel write - 开始第一章写作
```

**注意**：必须先完成前5步（constitution → specify → clarify → plan → tasks）才能开始写作。
